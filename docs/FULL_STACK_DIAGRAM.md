## SYSTEM OVERVIEW

Taproot is an AI-native, multi-tenant restaurant POS delivered as a monorepo (npm workspaces):

```
                         ┌─────────────────────────────────────────────┐
   Browser / iPad / PWA  │  Vercel  —  apps/web  (React SPA + PWA SW)    │
   ───────────────────►  │  static build (Vite), SPA rewrite to /index   │
                         └───────────────┬─────────────────────────────┘
                                         │ HTTPS  (VITE_API_URL)
                                         ▼
                         ┌─────────────────────────────────────────────┐
                         │  Railway  —  apps/api  (Fastify, Node 20)     │
                         │  node apps/api/dist/index.js  (port 8080)     │
                         └───┬───────────────┬───────────────┬──────────┘
                             │               │               │
                   ┌─────────▼───┐   ┌───────▼──────┐  ┌──────▼─────────────┐
                   │ PostgreSQL  │   │   Redis      │  │ External services  │
                   │ (Railway)   │   │  (Railway)   │  │ Stripe · Anthropic │
                   │ pg Pool     │   │ ioredis      │  │ Twilio · SMTP/SES  │
                   │ 47 tables   │   │ cache+pubsub │  │ Sentry · Plausible │
                   └─────────────┘   └──────────────┘  └────────────────────┘
```

- **Monorepo:** `apps/api` (Fastify backend), `apps/web` (React frontend), `packages/shared`
  (`@taproot/shared` — shared TypeScript types, built first in every pipeline).
- **Live:** frontend `https://taproot-pos.com` (Vercel) · API
  `https://taproot-production-3d63.up.railway.app` · health `GET /api/health`.
  *(? `railway.json` lists serviceDomain `taproot-production.up.railway.app:8080`; the live
  health-checked host is `taproot-production-3d63…` — Railway dashboard is source of truth.)*

## INFRASTRUCTURE & DEPLOYMENT

| Concern | Detail |
|---|---|
| Frontend host | **Vercel** — `apps/web/vercel.json`: build `@taproot/shared` then `@taproot/web`, output `apps/web/dist`, SPA rewrite `/(.*) → /index.html`, immutable asset cache, security headers (nosniff, X-Frame-Options DENY, Referrer-Policy) |
| Backend host | **Railway** — `railway.json` / `nixpacks.toml`: Node 20, `npm ci`, build shared+api, start `node apps/api/dist/index.js`, restart-on-failure (3 retries) |
| Database | PostgreSQL (Railway), accessed via `pg` Pool (`apps/api/src/db/client.ts`), **no ORM** |
| Cache / realtime | Redis (Railway) via `ioredis` — caching + pub/sub |
| Migrations | `node-pg-migrate`, `migrations/` (001–022), run manually in Railway console |
| Auto-deploy | push to `main` → Railway (API) + Vercel (web) redeploy |
| CI | GitHub Actions (api typecheck+tests, web typecheck+build, ESLint, secret scan) |

**Wiring caveat (SESSION_GUIDELINES):** `apps/web/.env.production` `VITE_API_URL` must equal the
live Railway host; CORS for `taproot-pos.com`/`www` is hardcoded in `index.ts`.

## REQUEST LIFECYCLE (backend)

`apps/api/src/index.ts` builds one Fastify instance with this order:

```
Request
  │
  ├─ helmet (CSP, HSTS, frameguard, referrer-policy)        [security headers]
  ├─ CORS (taproot-pos.com/www, *.vercel.app, localhost, env origins)
  ├─ @fastify/rate-limit (global 200/min/IP; stricter per-route)
  ├─ validation hooks (XSS/body/UUID — middleware/validation.ts)
  ├─ HTTPS redirect (prod, via x-forwarded-proto)
  ├─ Sentry request hooks
  │
  ├─ route plugins registered (auth + ~31 domain route files + admin)
  │
  ├─ GLOBAL preHandler:
  │     routeKey = `${method} ${routeOptions.url}`
  │     if routeKey ∈ PUBLIC_ROUTES        → allow (login, register, webhooks, /metrics, public menu…)
  │     if url startsWith /api/v1/admin/    → allow (admin routes self-auth via adminAuth)   [added 2026-06-09]
  │     else  fastify.authenticate()  →  checkSubscription()
  │
  └─ handler → service(s) → pg Pool / Redis / external API → reply
        errors → middleware/errorHandler.ts (production-safe)
```

## BACKEND ARCHITECTURE (`apps/api/src`)

**Stack:** Fastify v4 + TypeScript (strict). Deps incl. `@fastify/{cors,helmet,rate-limit,multipart,websocket}`,
`pg`, `ioredis`, `bull` (queues), `jsonwebtoken`, `bcrypt`, `otplib` (TOTP), `stripe`,
`@anthropic-ai/sdk`, `nodemailer`, `pdf-parse`, `csv-parse`, `zod`, `prom-client`, `@sentry/node`.
*(? `@apollo/server`/`graphql` are dependencies but no Apollo/`/graphql` registration was found in
`index.ts` — appears unused/not wired.)*

### Auth subsystem (`apps/api/src/auth/`)
`jwt.ts`, `permissions.ts` (RBAC), `audit.ts`, `crypto.ts` (AES-256-GCM), `schemas.ts` (zod),
`middleware.ts`, `routes.ts`. **Two independent JWT realms:**
- **Org JWT** (`JWT_SECRET`, HS256, 15m access / 30d refresh w/ rotation) — `fastify.authenticate`,
  roles `owner|manager|cashier|kitchen|readonly`, PIN login, MFA/TOTP, account lockout, refresh-reuse detection.
- **Admin JWT** (`ADMIN_JWT_SECRET`, HS256, issuer `taproot-admin`, aud `taproot-admin-api`) —
  `middleware/adminAuth.ts`, roles `super_admin|support|read_only`, session table check.

### Middleware (`apps/api/src/middleware/`)
`adminAuth.ts` · `errorHandler.ts` · `subscription.ts` (trial/subscription gate) · `validation.ts`.

### Routes (`apps/api/src/routes/` — 31 files) by domain
- **Auth/registration:** `auth/routes.ts` (login, login/mfa, login/pin, refresh, password reset),
  `registration.routes.ts` (`/register`, `/register/check-email`)
- **Catalog:** `inventory.routes.ts` (products, variants, categories, stock, archived, barcode, recipes,
  variance), `modifier.routes.ts`, `settings.routes.ts` (business, tax, receipt, payments, loyalty,
  online-ordering, dashboard-layout)
- **Sales:** `order.routes.ts` (`/locations/:id/orders` + void/refund/park/resume/split/merge/tip/receipt),
  `payment.routes.ts`, `customer.routes.ts`, `discount.routes.ts`, `cashDrawer.routes.ts`,
  `report.routes.ts` (dashboard, sales, end-of-day, tips, heatmap)
- **Table service:** `table.routes.ts` (floor plan, status), `kitchen.routes.ts` (KDS), `reservation.routes.ts`
- **Online/text:** `public.routes.ts` (no-auth `/public/:slug/menu` + order + pay), `sms.routes.ts` (Twilio)
- **AI/analytics:** `ai.routes.ts`, `intelligence.routes.ts` (forecast/staffing/menu/food-cost/feed),
  `analytics.routes.ts` (cohort, menu-engineering, peak-hours, staff-performance, customer-insights)
- **Import/migration:** `import.routes.ts` (upload→parse→confirm), `migration.routes.ts`
- **Growth/billing:** `billing.routes.ts` (Stripe subscription/portal), `onboarding.routes.ts`,
  `referral` (via service), `franchise.routes.ts`, `integrations.routes.ts` (QuickBooks/Xero export)
- **Platform:** `apiKeys.routes.ts`, `webhooks.routes.ts` (CRUD) + `webhook.routes.ts` (Stripe inbound),
  `scheduling.routes.ts` (timeclock + schedules), `websocket.routes.ts`
- **Admin/executive portal:** `admin.routes.ts` — `POST /admin/auth/login` (public), logout, `GET
  /admin/metrics`, `/admin/organizations(/:id)`, `PATCH /admin/organizations/:id`,
  `POST /admin/organizations/:id/impersonate`, `POST /admin/helpdesk/query`, `GET /admin/helpdesk/tickets`

### Services (`apps/api/src/services/` — ~45 files)
admin · ai · aiForecast · analytics · apiKey · cashDrawer · category · customer · discount ·
documentParser (Claude PDF/CSV menu parse) · email · employee · foodCost · forecast · franchise ·
giftcard · helpdesk · importJob · intelligence · inventory · kitchen · location · loyalty ·
migration · modifier · order · payment · product · public · purchaseOrder · **realtime** (Redis

## DATABASE (PostgreSQL — 47 tables across 22 migrations)

`001_initial_schema.js` creates ~35 core tables; 002–022 extend. All tenant tables carry
`organization_id` (multi-tenant isolation). Grouped:

- **Tenancy/identity:** organizations, locations, employees, refresh_tokens, password_reset_tokens,
  mfa_backup_codes, partner_codes, referral *(via service)*
- **Catalog:** products, product_variants, product_prices, categories, modifier_groups, modifiers,
  product_modifier_groups, suppliers, recipes, recipe_lines
- **Inventory:** inventory_levels, inventory_movements, purchase_orders, purchase_order_lines,
  variance_reports, variance_report_lines
- **Sales:** orders, order_line_items, payments, organization_order_sequences, applied_discounts,
  discounts, gift_cards, gift_card_transactions, loyalty_transactions
- **Table service:** tables, reservations
- **Cash/terminal:** cash_drawer_sessions, cash_drops, terminal_readers
- **Growth/platform:** subscriptions *(via 009)*, api_keys, webhooks, import_jobs, schedules,
  time_clock_entries, audit_logs (+ partitions), franchise columns (017)
- **Admin/helpdesk (022):** admin_users, admin_sessions, admin_impersonation_log,
  helpdesk_tickets, helpdesk_messages

Key facts (ground truth): `order_line_items` (not order_items), `product_prices` (not prices),
employees use `deleted_at` (no is_active) + `location_ids uuid[]` (no junction table),
`locations.tax_config` JSONB (no tax_rates table), prices stored in **integer cents**.

## KEY DATA FLOWS

### A. Auth (org)
```
LoginPage → POST /api/v1/auth/login → auth.service (bcrypt verify, issue org JWT + refresh)
  → store tokens in localStorage → apiFetch attaches Bearer → 401 ⇒ POST /auth/refresh (rotate) ⇒ retry
```

### B. POS cash sale
```
POSLayout (GET /products, /categories — React Query) → tap product (ModifierSheet if groups)
  → pos.store cart → PaymentSheet "Charge"
  → POST /locations/:id/orders {orderType, lineItems[]}  (order.service: tax from locations.tax_config)
  → POST /locations/:id/orders/:id/payments {paymentMethod:cash, amount, cashTendered}
       (payment.service: change_due, loyalty accrual; realtime.publishOrderEvent → Redis → KDS)
  → navigate /receipt (renders pos.store.lastCompletedOrder, enriches via GET /orders/:id/receipt)
```

### C. AI menu import
```
ImportPage upload (multipart) → POST /api/v1/imports/upload → import_job (status processing)
  → bull queue → documentParser.service (Claude parse PDF/CSV → cents via normalizeMenuPrice)
  → status awaiting_confirmation → ImportReview (edit prices) → POST /imports/:id/confirm {confirmedItems}
  → importJob.confirmImportJob → ProductSvc.createProduct (product + Default variant + price, org-scoped)
  → status completed → ImportReview invalidates ['products']/['categories'] → POS shows items
```

### D. Admin / executive portal
```
AdminLoginPage → POST /api/v1/admin/auth/login (PUBLIC; org-guard exempt) → admin.service.adminLogin
  (bcrypt, ADMIN_JWT_SECRET, admin_sessions row) → adminAuth.store
  → admin routes (authenticateAdmin) → metrics / organizations / impersonate (issues short-lived
    ORG JWT signed w/ JWT_SECRET so existing org middleware accepts it) / helpdesk (Claude)
```

### E. Realtime / channels
- Order events: service → Redis pub/sub → `websocket.routes.ts` → Kitchen Display / online-order bell
- Customer display: POS → BroadcastChannel (`displayChannel.ts`) → `/display` (no backend)
- Offline: PaymentSheet offline → IndexedDB queue (`offlineQueue.ts`) → replays on reconnect
